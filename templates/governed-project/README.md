## Conformance

`[project-name]` conforms to Lithos `[1.x]` as a full governed project. It declares that claim in an optional machine-readable adoption manifest (`[lithos-adoption-manifest.json]`) — the version and depth claimed, role holders, gate operation, and autonomous PR policy. The manifest is a declaration, not an authorization, and is kept vendor-neutral and portable.

## Boundaries

Ordinary development approval does not approve live/default-on behavior, external delivery, publishing, production configuration mutation, destructive actions, or autonomous operation. Higher gates in [`docs/AI_FLOW.md`](docs/AI_FLOW.md) still apply. Agents may open and update their own pull requests, but never self-approve, self-merge, enable ownerless auto-merge, or delete branches without explicit owner approval.
