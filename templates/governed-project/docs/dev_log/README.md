# Development log directory

Use this directory for task-level evidence logs. A dev log may record background, scope, decisions, verification commands, review outcomes, and follow-up tails.

Naming convention:

```text
docs/dev_log/YYYY-MM-DD-topic.md
```

Dev logs are evidence artifacts. They do not redefine product requirements above `GOAL.md`, PRD, design, roadmap, or current status.
